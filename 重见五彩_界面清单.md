# 《重见五彩》界面清单

> 版本：v0.1  
> 日期：2026-06-16  
> 用途：明确第一版灰盒原型需要哪些界面、每个界面的功能和最小交互。  
> 原则：第一版只做闭环必需界面，不做完整游戏 UI。

## 1. 界面总览

第一版只做 7 个界面：

| 编号 | 界面 | 是否必做 | 作用 |
| --- | --- | --- | --- |
| UI-01 | 开始界面 | 是 | 进入游戏 |
| UI-02 | 工坊界面 | 是 | 承载任务、角色提示和当前状态 |
| UI-03 | 剧情对白界面 | 是 | 展示短对白和推进剧情 |
| UI-04 | 工艺参数界面 | 是 | 选择釉料、罩釉、升温、氧化 |
| UI-05 | 开窑结果界面 | 是 | 展示成功或失败 |
| UI-06 | 失败复盘界面 | 是 | 显示原因、建议和知识解锁 |
| UI-07 | 展柜与档案界面 | 是 | 展示最终作品和解锁档案 |

暂不做：

- 设置界面
- 存档界面
- 背包
- 大地图
- 完整图鉴
- 角色详情页
- 商店
- 复杂订单列表

## 2. UI-01 开始界面

### 目标

让玩家进入游戏，并建立题材印象。

### 显示内容

- 游戏标题：《重见五彩》
- 副标题：在 1950 年代的醴陵，抢救濒临断代的釉下五彩技艺。
- 按钮：开始

### 交互

- 点击“开始”进入工坊界面。

### 灰盒要求

- 纯文字即可。
- 背景图可后续替换。

## 3. UI-02 工坊界面

### 目标

作为第一版主界面，展示当前任务和下一步行动。

### 显示内容

- 当前任务标题
- 当前任务说明
- 简单资源状态
- 角色提示
- 主操作按钮

### 第一版资源状态

只显示 3 项即可：

| 资源 | 用途 |
| --- | --- |
| 时间 | 表示试验消耗 |
| 资金 | 表示不能无限试烧 |
| 资料完整度 | 表示工艺理解进度 |

第一版不需要真实复杂计算，可以只做展示或轻量变化。

### 交互

- 开始第一次试烧
- 查看当前提示
- 进入参数选择

### 灰盒要求

- 一屏内完成。
- 不做复杂拖拽。

## 4. UI-03 剧情对白界面

### 目标

用短对白推动流程，不用长篇说明。

### 显示内容

- 角色名
- 角色对白
- 下一句按钮
- 跳过按钮，可后续再加

### 第一版对白节点

| 节点 | 触发时机 |
| --- | --- |
| 开场对白 | 点击开始后 |
| 第一次发灰后 | 第一次开窑结果 |
| 二烧失败后 | 第一次二烧失败 |
| 成功后 | 二烧成功 |

### 灰盒要求

- 每段对白不超过 3 句。
- 每句尽量不超过 40 个汉字。

## 5. UI-04 工艺参数界面

### 目标

让玩家选择影响开窑结果的关键变量。

### 显示内容

| 参数 | 控件 | 可选值 |
| --- | --- | --- |
| 釉料 | 单选按钮 | 旧石灰青釉 / 长石釉 |
| 罩釉厚度 | 单选按钮 | 薄 / 中 / 厚 |
| 升温节奏 | 单选按钮 | 慢 / 中 / 快 |
| 氧化气氛 | 单选按钮 | 不足 / 普通 / 充分 |

### 辅助内容

- 角色建议
- 当前目标
- 入窑烧成按钮

### 交互

- 玩家选择参数。
- 点击“入窑烧成”。
- 系统调用结果判定。

### 灰盒要求

- 先用单选按钮，不做滑条。
- 每个参数旁边只放一句短说明。

## 6. UI-05 开窑结果界面

### 目标

提供情绪反馈：失败时有问题，成功时有成就感。

### 显示内容

- 结果标题
- 作品图或占位图
- 结果描述
- 评级
- 主按钮

### 结果类型

| 结果 | 评级 | 下一步 |
| --- | --- | --- |
| 颜色发灰 | 可用但不理想 | 查看问题 |
| 黑壳 / 发乌 | 失败 | 进入复盘 |
| 针孔 / 气泡 | 失败 | 进入复盘 |
| 基本可用 | 可用 | 继续优化 |
| 五彩重现 | 成功 | 进入展柜 |

### 交互

- 查看复盘
- 再试一次
- 进入展柜

### 灰盒要求

- 第一版可用色块或文字代替图片。
- 成功和失败必须明显不同。

## 7. UI-06 失败复盘界面

### 目标

让玩家理解失败原因和下一步调整方向。

### 显示内容

```text
现象：
原因：
建议：
解锁知识：
```

### 示例

```text
现象：釉面出现针孔和气泡。
原因：罩釉过厚且升温太快，气体来不及排出。
建议：降低罩釉厚度，放慢升温。
解锁知识：薄罩釉、有机物排出窗口。
```

### 交互

- 调整方案再试
- 查看档案，可后续再加

### 灰盒要求

- 复盘信息必须直指可调整参数。
- 不写成百科长文。

## 8. UI-07 展柜与档案界面

### 目标

作为第一版结尾，给玩家一个明确完成感。

### 显示内容

- 成功作品图或占位图
- 作品名：《重见五彩》釉下五彩瓶
- 简短说明
- 解锁档案列表
- 重新开始按钮

### 解锁档案

第一版只放 3 条：

1. 三烧改二烧
2. 长石釉
3. 失败复盘记录

### 结尾文案

> 这不是一个人的发明，而是一代人把快要断掉的知识重新变成可以传下去的能力。

### 灰盒要求

- 必须有明确结尾。
- 允许重新开始。

## 9. 第一版页面流转

```text
开始界面
  ↓
工坊界面
  ↓
剧情对白
  ↓
工艺参数界面
  ↓
开窑结果界面
  ↓
失败复盘界面
  ↓
工艺参数界面
  ↓
开窑结果界面
  ↓
展柜与档案界面
```

## 10. 验收标准

- 7 个界面都可以访问。
- 每个界面只有一个主要行动。
- 玩家不会不知道下一步点哪里。
- 失败复盘能回到参数界面。
- 成功结果能进入展柜。
- 重新开始可回到开始界面。
