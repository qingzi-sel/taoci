# 《重见五彩》失败规则表

> 版本：v0.1  
> 日期：2026-06-16  
> 用途：为第一版灰盒原型提供成功、失败、原因和复盘建议的规则依据。  
> 原则：先用简单条件判断，不做复杂真实陶瓷模拟。

## 1. 规则目标

本规则表服务一个核心体验：

> 玩家不是靠随机试错成功，而是通过失败复盘理解下一步该调整什么。

第一版不追求真实窑炉模拟，只追求清晰因果：

- 为什么失败。
- 失败暴露了哪个工艺问题。
- 玩家下次应该改什么。
- 改对以后为什么能成功。

## 2. 初版输入参数

| 参数 ID | 中文名 | 可选值 | 说明 |
| --- | --- | --- | --- |
| glazeType | 釉料 | oldLime / feldspar | 旧石灰青釉 / 长石釉 |
| coverThickness | 罩釉厚度 | thin / medium / thick | 薄 / 中 / 厚 |
| heatingSpeed | 升温节奏 | slow / normal / fast | 慢 / 中 / 快 |
| oxygenLevel | 氧化气氛 | low / normal / high | 不足 / 普通 / 充分 |

## 3. 输出结果结构

程序中建议每次开窑返回一个 result 对象。

```js
{
  id: "pinholes",
  title: "针孔与气泡",
  description: "釉面出现细小针孔和局部气泡，作品不能进入展柜。",
  cause: "罩釉过厚且升温过快，排气窗口不足。",
  suggestion: "降低罩釉厚度，并放慢升温节奏。",
  isSuccess: false,
  archiveUnlock: ["失败档案：针孔与气泡"]
}
```

## 4. 结果优先级

当多个失败条件同时满足时，第一版按以下优先级返回结果：

1. 黑壳 / 发乌
2. 针孔 / 气泡
3. 颜色发灰
4. 留白不清
5. 周期过长
6. 普通可用
7. 成功

优先级的原因：

- 黑壳、针孔、气泡更适合作为二烧失败的强反馈。
- 发灰适合作为第一次旧釉问题。
- 留白不清后续接入分水小游戏时再强化。

## 5. 初版判定规则

### 5.1 成功

| 条件 | 结果 |
| --- | --- |
| 长石釉 + 薄罩釉 + 慢升温 + 氧化充分 | 稳定二烧成功 |

结果文案：

- 标题：五彩重现
- 描述：底色洁净，色彩明亮，釉面较稳定，未见明显黑壳、针孔和气泡。
- 原因：长石釉提供更洁净的底色，薄罩釉和慢升温保留了有机物排出窗口，氧化充分帮助烧成稳定。
- 建议：记录本次方案，进入展柜与档案解锁。

### 5.2 黑壳 / 发乌

| 条件 | 结果 |
| --- | --- |
| 氧化不足 + 厚罩釉 | 黑壳 |
| 氧化不足 + 快升温 | 发乌 |

结果文案：

- 标题：黑壳与发乌
- 描述：局部画面发乌，釉面下出现黑色或暗色痕迹。
- 原因：有机物排出不充分，且氧化气氛不足。
- 建议：提高氧化充分度，降低罩釉厚度，避免过快升温。

解锁档案：

- 失败档案：黑壳
- 工艺档案：有机物排出窗口

### 5.3 针孔 / 气泡

| 条件 | 结果 |
| --- | --- |
| 厚罩釉 + 快升温 | 针孔与气泡 |
| 厚罩釉 + 中等升温 | 气泡 |

结果文案：

- 标题：针孔与气泡
- 描述：釉面出现细小针孔和局部气泡，作品不能进入展柜。
- 原因：罩釉过厚，升温节奏偏快，气体来不及排出。
- 建议：改用薄罩釉或中等罩釉，并放慢升温。

解锁档案：

- 失败档案：针孔与气泡
- 工艺档案：薄罩釉

### 5.4 颜色发灰

| 条件 | 结果 |
| --- | --- |
| 旧石灰青釉 | 颜色发灰 |

结果文案：

- 标题：颜色发灰
- 描述：作品成形，但釉下色显得灰暗，不够洁净明亮。
- 原因：旧石灰青釉底色偏青偏暗，影响釉下彩料呈色。
- 建议：尝试长石质白釉，并建立试片记录。

解锁档案：

- 工艺档案：长石釉
- 失败档案：颜色发灰

### 5.5 留白不清

第一版可作为低频失败或后续分水小游戏结果。

| 条件 | 结果 |
| --- | --- |
| 后续接入分水小游戏时，分水操作评分低 | 留白不清 |

结果文案：

- 标题：留白不清
- 描述：纹样边界不够清楚，部分色料越界。
- 原因：墨线控制或分水导流不稳定。
- 建议：练习双勾分水，控制水量和导流速度。

解锁档案：

- 工艺档案：双勾分水
- 失败档案：留白不清

### 5.6 周期过长

| 条件 | 结果 |
| --- | --- |
| 使用三烧制流程 | 周期过长 |

结果文案：

- 标题：周期过长
- 描述：作品可以完成，但烧制周期长，成本和窑炉占用较高。
- 原因：传统三烧制需要额外低温焙烧，流程更稳但耗时更长。
- 建议：在理解旧流程后，尝试研发二烧制。

解锁档案：

- 工艺档案：三烧制
- 工艺档案：二烧制研发

### 5.7 普通可用

| 条件 | 结果 |
| --- | --- |
| 长石釉 + 中罩釉 + 中升温 + 普通氧化 | 普通可用 |
| 长石釉 + 薄罩釉 + 中升温 + 普通氧化 | 基本可用 |

结果文案：

- 标题：基本可用
- 描述：作品质量比第一次更好，但稳定性仍不足。
- 原因：关键方向正确，但烧成控制还不够充分。
- 建议：继续优化升温节奏和氧化气氛。

## 6. 推荐初版伪代码

```js
function judgeKilnResult(params) {
  const { glazeType, coverThickness, heatingSpeed, oxygenLevel } = params;

  if (
    glazeType === "feldspar" &&
    coverThickness === "thin" &&
    heatingSpeed === "slow" &&
    oxygenLevel === "high"
  ) {
    return results.success;
  }

  if (oxygenLevel === "low" && coverThickness === "thick") {
    return results.blackShell;
  }

  if (oxygenLevel === "low" && heatingSpeed === "fast") {
    return results.darkened;
  }

  if (coverThickness === "thick" && heatingSpeed === "fast") {
    return results.pinholes;
  }

  if (coverThickness === "thick") {
    return results.bubbles;
  }

  if (glazeType === "oldLime") {
    return results.grayColor;
  }

  return results.usable;
}
```

## 7. 复盘提示模板

每个失败结果应显示四段信息：

```text
现象：
你看到了什么问题。

可能原因：
这个问题和哪些工艺变量有关。

下一步建议：
玩家下一次应该调整什么。

知识解锁：
这次失败让玩家理解了什么工艺概念。
```

## 8. 首版验收标准

- 至少有 5 种失败结果。
- 每种失败都有原因和建议。
- 成功条件明确。
- 玩家能通过复盘找到成功组合。
- 规则可以直接写进 JavaScript。

## 9. 后续扩展方向

灰盒跑通后再考虑：

- 增加随机波动，但不能覆盖核心因果。
- 增加角色专长对成功率的影响。
- 增加试片系统，降低未知结果概率。
- 增加分水小游戏评分。
- 增加选瓷评级。
- 增加订单对品质和周期的要求。
